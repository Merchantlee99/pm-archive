# TPM Field Guide IA와 페이지 맵

## 1. 전체 사이트 맵
- `/`
- `/projects`
- `/projects/[id]`
- `/writing`
- `/writing/[id]`
- `/about`
- `/contact`
- `/signals`

## 2. 전역 내비게이션
상단 네비게이션은 아래 4개만 유지한다.
- 글
- 프로젝트
- 소개
- 연락

`Signals`는 보조 아카이브이므로 헤더에는 넣지 않는다. 푸터와 관련 모듈 링크로만 접근한다.

## 3. 각 페이지 역할
### Home
최신 글과 대표 사례가 함께 보이는 개인 아카이브 홈

### Projects
결과물이 아니라 `판단이 남은 사례집`

### Writing
블로그가 아니라 `기준 아카이브`

### About
이력 요약이 아니라 `왜 TPM으로 가는지 설명하는 페이지`

### Contact
가볍고 명확한 연결 포인트

### Signals
외부 변화를 기록하는 보조 노트

## 4. 홈 IA
고정 순서:
1. Home intro
2. Lead article
3. Latest writing rows
4. Sidebar about
5. Sidebar project
6. Sidebar links / tags
7. Footer

## 5. 페이지 간 연결 원칙
- Home에서 가장 먼저 `최신 글`로 신뢰를 만든다
- 프로젝트는 홈 우측 레일의 대표 사례로 보여준다
- `About`은 우측 레일과 별도 페이지에서 보완한다
- `Signals`는 홈 흐름을 방해하지 않는다

## 6. 콘텐츠 밀도 우선순위
무게 순서:
1. Lead article
2. Writing rows
3. Sidebar project
4. Sidebar about
5. Projects list
6. Signals

## 7. 홈에서 제외할 것
- Signals teaser
- 모든 프로젝트를 한 번에 보여주는 grid
- 긴 자기소개
- KPI 과시용 숫자 카드
- large hero stage

## 8. Secondary Page 레이아웃 원칙
### Projects Index
- lead case 1개
- supporting rows 2~4개
- grid보다 row 위주

### Project Detail
- hero/title
- thesis
- sections as document flow
- card cluster 금지

### Writing Index
- lead article 1개
- 나머지 row list
- category는 보조 strip 정도

### About
- intro
- current focus
- how I work
- toward TPM

### Contact
- 짧은 설명
- 핵심 연락 수단
- 불필요한 장식 없음

### Signals
- source / title / short interpretation row

## 9. IA 체크 기준
- 홈에서 5초 안에 방향이 읽히는가
- 최신 글이 첫 증거로 보이는가
- 우측 레일 프로젝트가 과하지 않은가
- 소개가 보조 모듈로 자연스럽게 읽히는가
- Signals가 메인 흐름을 흐리지 않는가
