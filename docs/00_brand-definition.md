# TPM Field Guide 브랜드 정의

## 1. 프로젝트 한 줄 정의
이 사이트는 `글과 프로젝트를 통해 판단의 구조를 남기며 TPM으로 수렴해가는 사람의 개인 아카이브`다.

## 2. 왜 이 콘셉트인가
기존 홈은 브랜드 랜딩에 가까웠고, 사용자가 글과 사례를 자연스럽게 읽는 흐름이 약했다.  
이번 리빌드는 `가이드북 / 사례집 / 기록집` 중에서도 특히 `기록이 먼저 보이는 아카이브`의 감각으로 옮긴다.

즉, 이 사이트는 다음 둘을 동시에 만족해야 한다.
- `정리된 기록자`처럼 보여야 한다
- `읽기 쉬운 에디토리얼 포트폴리오`처럼 보여야 한다

## 3. 사이트가 보여줘야 하는 사람상
- 글로 기준을 남기고 프로젝트로 증명하는 사람
- 고객의 언어를 문제 정의로 번역하는 사람
- 화면보다 구조를 먼저 읽는 사람
- 비전공자에서 출발했지만 TPM으로 분명하게 향하는 사람

## 4. 사용자에게 남겨야 하는 인상
- 기록된 판단을 남기는 사람
- 구조화된 사례를 만드는 사람
- 기술 감각은 있지만 과장하지 않는 사람
- 읽기 편한 아카이브를 가진 사람

## 5. 핵심 브랜드 문장
메인 브랜드 문장:

`고객과 제품, 프로젝트와 판단을 기록하며 TPM으로 가는 사람의 필드 가이드`

홈용 짧은 버전:

`기록하고, 구조화하고, 사례로 남깁니다`

## 6. 시각적 톤의 기준
- 메인 톤: `Plain Editorial`
- 보조 재질: `Cool Paper`
- 화면 인상: 정리된 문서와 아카이브 레일이 먼저 보이는 느낌
- 전반 감정: 조용함, 선명함, 오래 봐도 질리지 않음

## 7. 반드시 버릴 것
- orbit, comet, radar, 우주 메타포
- control room 과장
- hero stage 과시
- 큰 둥근 카드 스택
- glass를 주인공처럼 쓰는 방식
- 대시보드형 KPI 과시

## 8. 반드시 유지할 것
- 기술적 정밀함
- 시스템적 사고
- 큰 한국어 타이포
- 읽기 쉬운 위계
- 최신 글과 대표 프로젝트의 균형

## 9. 이 사이트의 첫인상 목표
첫 5초 안에 사용자가 아래를 이해해야 한다.
- 이 사람은 글을 중심으로 기준을 남긴다
- 실제 프로젝트 사례도 갖고 있다
- TPM으로 가는 방향이 분명하다

## 10. 구현 전 브랜드 체크 질문
새 화면이나 섹션을 만들 때마다 아래를 체크한다.
- 이게 “기록된 판단”처럼 보이는가?
- 너무 테크 데모 같지는 않은가?
- 글과 프로젝트 중 하나만 과하게 무겁지 않은가?
- 장식보다 위계가 먼저 읽히는가?
