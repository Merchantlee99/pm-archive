# 홈 와이어프레임

## 1. 홈 목표
홈은 “브랜드 랜딩”이 아니라,  
`최신 글 → 대표 사례 → 보조 정보` 순서로 읽히는 개인 아카이브여야 한다.

## 2. First View 구조
### Desktop
- 좌측 64%
  - 사이트 소개
  - 홈 제목
  - 짧은 설명
  - lead article
- 우측 36%
  - about module
  - featured project module
  - links / tags

### Mobile
- 소개
- lead article
- 최신 글 rows
- about / project / links 순서로 stack

## 3. Intro 카피 구조
- line 1: 사이트 정체성
- line 2: 무엇을 남기는지
- 설명 1문장

## 4. 배경 구조
- 종이 같은 밝은 배경
- 얇은 divider와 rail만
- 장식보다 문서 구조가 먼저 보여야 함
- orbit, stage lighting, radial wash 금지

## 5. Lead Article
홈에서 가장 먼저 나오는 증거 영역

### 목적
- “이 사람은 생각이 있다”를 가장 먼저 보여줌
- 글을 card가 아니라 큰 아카이브 항목처럼 보여줌

### 구성
- 날짜 / 카테고리
- 제목
- 2~3줄 설명
- 태그

## 6. Featured Project Module
홈에서는 우측 레일 안의 보조 모듈

### 목적
- 글의 기준이 실제 사례로 이어진다는 것을 간결하게 보여줌

### 구성
- 프로젝트명
- 한 줄 설명
- 역할
- 핵심 변화 1줄
- 링크

## 7. Latest Writing Rows
### 목적
- lead article 아래에서 최근 기록 흐름을 보여줌

### 구성
- 날짜
- 제목
- 짧은 설명
- 태그

## 8. Sidebar Modules
- About
- Featured Project
- Links
- Tags / Signals

## 9. Footer
- 브랜드 한 줄
- 핵심 링크
- copyright

## 10. 세로 리듬
- Intro: 짧고 명확해야 함
- Lead article: 가장 강한 증거
- Writing rows: 반복되는 기본 리듬
- Sidebar: 조용한 보조 레일

## 11. 패턴 매핑
- Main column: `lead item + row list`
- Sidebar: `rail modules`
- Footer: `plain link row`

## 12. 홈용 체크포인트
- hero처럼 보이면 실패
- ordinary card grid처럼 보이면 실패
- 프로젝트 모듈이 광고처럼 보이면 실패
- 글 리스트가 단순 블로그 인덱스처럼만 보이면 실패

## 13. 화면 판단 기준
- Home이 개인 포트폴리오 아카이브처럼 보이는가
- 최신 글이 첫 증거로 읽히는가
- 프로젝트 모듈이 과하지 않은가
- 정보가 많아도 읽기 쉬운가
